# 📘 5분 AI – Product Requirements Document (PRD)

## 1. 제품 개요 (Overview)

**제품명:** 5분 AI (Five Minutes AI)

**형태:** 토스 앱인토스 플랫폼용 WebView 기반 AI 마이크로러닝 앱

**목적:** AI 입문자(직장인·시니어·경단녀)가 하루 5분 학습으로 AI 활용 역량을 자연스럽게 쌓고, 챗사피엔스 교육 생태계로 유입될 수 있는 신규 진입 채널 확보

**핵심 가치:** 짧다, 쉽다, 바로 써먹을 수 있다.

---

## 2. 목표 (Goals)

### 🎯 Primary Goals

1. 토스 플랫폼에서 **12월 한 달간 10만/일 알림 노출** 효과를 극대화하여 신규 유입 확보
2. AI 초보자에게 부담 없는 학습 경험 제공 → ‘AI 접근성 개선’
3. AI 실습 경험 기반 긍정적 사용자 경험 → 챗사피엔스의 무료 공개특강/입문 과정 전환

### 🚫 Non-Goals

- 직접적인 판매/결제 유도 기능 없음
- LMS 대체 목적 아님
- 고급 AI 교육 플랫폼 기능 제공 아님

---

## 3. 핵심 가치 제안 (Value Proposition)

1. **하루 5분이면 AI를 배운다** – 초단기 학습 설계
2. **토스 사용자에게 최적화된 직관적인 UI**
3. **실전 중심 VOD + 챌린지 시스템**
4. **무료 공개특강으로 자연 유입되는 퍼널 구조**
5. **시니어/경단녀도 가능한 ‘기초 친화 UX’**

---

## 4. 타깃 사용자 (Persona)

### 👤 Persona A — 직장인 30~50대

- 필요: 업무 효율 향상, 문서 자동화
- 어려움: 시간이 부족함, 무엇부터 배워야 할지 모름

### 👤 Persona B — 시니어 50~70대

- 필요: AI 첫 경험, 쉬운 설명, 작은 성공 경험
- 어려움: 기술 두려움, 어려운 용어

### 👤 Persona C — 경단녀·초보자

- 필요: 간단한 AI 실습, 5분 과제, 자동화 루틴
- 어려움: 진입장벽, 도구 선택 어려움

---

## 5. 사용자 시나리오 (User Story)

### Story 1: 직장인 김00

- 토스 알림 “5분 AI 무료 학습” 클릭
- 5분 VOD로 문서 자동화 배움
- 강의 완료 팝업 → 무료 공개특강 신청

### Story 2: 시니어 박00

- 이미지 생성 실습에서 첫 성공 경험
- 챌린지 완료 후 “AI 무료 공개특강” 신청

### Story 3: 경단녀 이00

- 데일리 챌린지로 매일 실습
- AI 자동화 루틴에 흥미 → 입문 강의로 전환

---

## 6. 기능 요구사항 (Feature Requirements)

### 6.1 홈 화면 기능

- 추천 강의 3개 표시 (동적 변경 가능)
- ‘오늘의 5분 강의’ 메인 CTA
- 14일 챌린지 카드 및 진행률 표시

### 6.2 강의 리스트 & 상세 기능

- 카테고리 필터(전체/기초/직장인/시니어)
- 섬네일 + 제목 + 난이도 + 5분 태그
- 광고 시청 후 강의 오픈
- 영상 플레이어(배속·전체화면·완료 버튼)

### 6.3 데일리 챌린지 기능

- 14일 미션 세트
- 결과물 업로드(텍스트/이미지)
- 완료 체크 → 배지/성취감 제공

### 6.4 마이페이지 기능

- 시청 완료율 표시
- 챌린지 진행률 표시
- 추천 강의 3개
- “무료 공개특강 안내” URL 배너

### 6.5 무료 공개특강 URL 연결

- settings.webinar_url 동적 연동
- 강의 완료 팝업 → 공개특강 이동
- 챌린지 완료 팝업 → 공개특강 이동
- 마이페이지 → 공개특강 배너 노출

### 6.6 관리자 설정 기능

- webinar_url (매월 변경 가능)
- webinar_title
- 히어로 배너 문구/버튼 텍스트

---

## 7. UI/UX 요구사항

### 공통

- 모바일 WebView 최적화 (토스 앱 기준)
- 최소한의 입력, 최대한의 시청·탐색 경험 제공
- 시니어 UX 고려: 큰 버튼·짧은 문장·명확한 CTA

### 화면 6종 (정의됨)

1. 홈(Home)
2. 강의 리스트(Course List)
3. 강의 상세(Course Detail)
4. 영상 플레이어(Video Player)
5. 데일리 챌린지(Daily Challenge)
6. 마이페이지(My Page)

---

## 8. 정보 구조 (Information Architecture)

```
홈
 ├─ 추천 강의
 ├─ 오늘의 5분 강의
 ├─ 14일 챌린지

강의
 ├─ 카테고리별 리스트
 └─ 강의 상세 → 광고 → 영상 플레이어

챌린지
 ├─ Day 리스트
 └─ Day 상세 → 업로드 → 완료

마이페이지
 ├─ 학습 기록
 ├─ 챌린지 현황
 └─ 무료 공개특강

```

---

## 9. 기술 요구사항 & 제한사항

- Type: WebView 기반 서비스
- Framework: Vibe(비브) 코딩툴 사용
- 광고 시청 → 강의 오픈 로직 필요
- 영상 플레이어는 Web 지원 가능한 플레이어 사용
- 외부 랜딩 URL은 정보 전달 목적이어야 함 (토스 정책 준수)

---

## 10. 토스 플랫폼 정책 고려사항

- 직·간접 결제 유도 금지
- 외부 링크는 “교육 목적/정보 제공” 용도로만
- 과도한 광고·수익성 문구 지양
- WebView 내 팝업/이벤트는 심플하게 유지

---

## 11. KPI / 성공 지표

### Primary KPI

- 12월 신규 방문자 수 (목표: 30만 이상)
- 강의 시청 완료율 (목표 40%)
- 챌린지 Day5 도달률 (목표 30%)
- 공개특강 랜딩페이지 클릭률 (목표 10%)

### Secondary KPI

- 특강 → 입문 강의 전환율
- 사용자 만족도 리뷰 수

---

## 12. 향후 확장 로드맵

### Phase 2 (1~2개월 후)

- 챌린지 30일 버전 추가
- 사용자 레벨/배지 시스템 강화
- SNS 공유 자동화 기능

### Phase 3 (3~6개월 후)

- AI 핸즈온 실습 자동화 모듈
- 챗사피엔스 통합 계정 연동
- 고급 과정 + AI 자동화 루틴 빌더 출시
